Document Object Model (DOM)

Writing Clean Code. The Document Object Model (DOM) is a programming interface for web documents. It represents the page so that programs can change the document structure, style, and content. The DOM represents the document as nodes and objects; that way, programming languages can interact with the page. A web page is a document that can be either displayed in the browser window or as the HTML source. In both cases, it is the same document but the Document Object Model (DOM) representation allows it to be manipulated. As an object-oriented representation of the web page, it can be modified with a scripting language such as JavaScript.

  • Basis of HTML5 is "New Features should be based on HTML, CSS the DOM and JavaScript..."
  • DOM provides common tree-like structure that all pages should follow
  • Computer Scientists love trees (the mathematical kind) because you can test them.
Very root of the tree is HTML tag. There are three parts to a well formed document.
  • DOCTYPE, version of HTML you will be using
  • Head, Metadata
  • Body, Displayable Content

Body is the bulk of the page. It is important to write "Clean Code" or well-formatted (tree-like) code so that it is easily read by different programs.



W3C WCAG 2.0

W3C WCAG 2.0/2.1 - World Wide Web Consortium Web Content Accessibility Guildlines

Principle, Not technology-based.

Defines how to make Web content more accessible to people with disabilities. Accessibility involves a wide range of disabilities, including visual, auditory, physical, speech, cognitive, language, learning, and neurological disabilities.

Four Principles of W3C WCAG (POUR):

  • Perceivable
  • Operatable
  • Understandable
  • Robust

Designing with accessibility in mind is the right thing to do. Adhere to standards and pay special attention to the semantics behind the HTML5 tags.



Validate Your Code

THIS IS VERY IMPORTANT!

Browsers can act like the "helicopter parents" of programming and they can fix errors in code that otherwise can go unchecked. As your pages become more complex "hidden" areas can be devastating to your webpage.

The W3C Markup Validation Service is a perfect tool to use to stick to the same World wide standards

Not only should you validate for syntax, but you should also validate for the accessibility of the semantics. You can use wave.webaim.org to validate for accessibility.

Funkify Disability Simulator is a great option to virtually check your website for accessibility by simulating different disabilities one may experience. Really cool site! Highly Recommend

Links/Anchors

HTTP protocol to inform the browser to load a Web page when you click a hyperlink
<a href="URL">link name</a>
Anchor tags can be used to link to a specific location within an HTML file (even within the same HTML file)
Firstly, a location must be defined using the tag: <a name="xxxx">link</a> with xxx being the location name.
A link to the location is created using the tag <a href="#xxx">link</a>. If the location is in another document, its file name too must be included as well: <a href="URL#xxx">link</a>
Link to the Study Guide I am referencing

Various other protocols may be used. For example, to create a link to an email address use the 'mailto' protocol. Note that this depends on the user's email programme being correctly configured, and so may not always work. However, this feature is commonly used on the Web to contact the webmaster or to get more information from sites.
The following anchor tag creates a mail link:
<a href="mailto:username@domainname">user</a>
A subject line for the email message can also be provided:
<a href="mailto:username@address.com?SUBJECT=e-mail from a friend">user</a>


Images


The <img> tag is used to embed an image into a Web document:
<img src="URL" alt="Alternate Text for Screenreaders/Accessibility">
The URL is the location of the image file. The ALT attribute specifies text to be displayed if the browser does not display the image It is very important to add alternative text to images that tell information to readers. If people have screenreaders, they may not be able to get access to information if they cannot see the image containing it.
In-line images are often placed inside anchors. The HTML code to create an image as a hyperlink is:
<a href="URL"><img src="URL" alt="text"></a>
